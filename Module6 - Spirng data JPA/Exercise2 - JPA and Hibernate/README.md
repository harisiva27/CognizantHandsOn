
# Module 6 - Spring Data JPA

## Mandatory Hands-on 2: Difference between JPA, Hibernate and Spring Data JPA

### Objective

The objective of this hands-on is to understand the differences between JPA, Hibernate, and Spring Data JPA, and to learn how they work together to simplify database operations in Java applications.

### Topics Covered

- Object-Relational Mapping (ORM)
- Java Persistence API (JPA)
- Hibernate Framework
- Spring Data JPA
- Relationship between JPA, Hibernate, and Spring Data JPA
- Advantages of using Spring Data JPA

### Key Differences

| Feature | JPA | Hibernate | Spring Data JPA |
|---------|-----|-----------|-----------------|
| Type | Specification | ORM Framework | Spring Module |
| Purpose | Defines persistence standards | Implements JPA and provides ORM features | Simplifies JPA-based database operations |
| Implementation | No | Yes | Uses JPA and Hibernate internally |
| Boilerplate Code | More | Moderate | Very Less |
| Usage | Defines APIs | Handles ORM | Provides repositories and query methods |

### Advantages of Spring Data JPA

- Reduces boilerplate code.
- Provides built-in CRUD operations.
- Supports query methods without writing SQL.
- Integrates seamlessly with Spring Boot.
- Improves developer productivity.

### Learning Outcome

After completing this hands-on, I learned:

- The concept of Object-Relational Mapping (ORM).
- The role of JPA as a persistence specification.
- How Hibernate implements JPA.
- How Spring Data JPA simplifies database access.
- The relationship between JPA, Hibernate, and Spring Data JPA.
